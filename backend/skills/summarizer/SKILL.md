# Summarizer — 概括者

## Overview

概括者专注于信息的结构化整理，将分散的内容提炼为清晰的概览。核心能力包括：

- 章节摘要：提炼章节核心事件和情绪脉络
- 人物志：整理角色的成长轨迹和关系变化
- 设定概览：梳理世界观的关键要素和规则
- 剧情时间线：按时间顺序整理事件发展

**核心原则**：好的概括不是缩写，是提炼关键信息。

## 触发条件

用户提及以下关键词：
- 「总结」「摘要」「概括」「提炼」
- 「人物志」「人物档案」「角色整理」
- 「设定概览」「世界观整理」「规则汇总」
- 「时间线」「剧情梳理」「事件整理」
- 「回顾」「前面讲了什么」

## 功能模块

### 模块 A：章节摘要

**输入**：一章或多章正文
**输出**：
- 一句话梗概
- 核心事件（发生了什么）
- 人物行动（谁做了什么）
- 情绪曲线（本章的情感走向）
- 关键信息（新出现的设定/伏笔）

### 模块 B：人物志更新

**输入**：章节内容 + 已有角色信息
**输出**：
- 角色状态变化
- 新获得的信息
- 关系变化
- 性格/动机的新展现

### 模块 C：设定概览

**输入**：世界观相关内容
**输出**：
- 核心规则清单
- 势力/组织关系图
- 地图/地点说明
- 力量/技术体系总结

### 模块 D：剧情时间线

**输入**：多章内容
**输出**：
- 按时间顺序的事件列表
- 伏笔设置与回收对照
- 各线并行发展情况

## 工作流程

### 单章摘要

1. 通读章节，标记关键事件
2. 识别情绪转折点
3. 提取人物行动和动机
4. 记录新信息和伏笔
5. 按格式输出

### 多章整合

1. 逐章生成摘要
2. 合并同类项，提炼主线
3. 整理人物状态变化
4. 梳理伏笔设置与回收
5. 生成时间线和概览

## 交互规范

### 需求模糊时

如果用户只说「帮我整理一下」：
1. 询问：「需要整理什么内容？（章节/人物/设定/时间线）」
2. 询问：「范围是？（单章/多章/全文）」

### 整理完成后

```
---
💡 **下一步可以：**
- 输入「导出记忆」保存到 .crazy_writer_memory/
- 输入「继续整理」处理更多章节
- 输入「生成大纲」基于整理结果规划后续
- 输入「人物关系图」可视化角色关系
```

## 使用场景

| 场景 | 功能 | 价值 |
|------|------|------|
| 长时间未续写 | 回顾摘要 | 快速回忆前文 |
| 多人协作 | 人物志同步 | 确保角色不 OOC |
| 中期检查 | 设定概览 | 发现设定矛盾 |
| 规划后续 | 时间线梳理 | 合理安排伏笔回收 |
| 完本整理 | 全文摘要 | 生成作品档案 |
